# Executive Summary — ERC EDW Data-Ingress Tier

**Situation.** The ERC EDW (Oracle Autonomous Data Warehouse on OCI, four-workstream medallion) needs a reliable tier that turns ~30 heterogeneous sources — vendor APIs, internal databases, and other California department systems — into clean, conformed, EDW-ready tables and lands them in the OCI staging zone. From there, Agiline's aiWorks platform picks up the staged data and loads it into the EDW, executing Billow's PL/SQL scripts; Billow then consumes the EDW data into Power BI. This tier is the "ingest → cleanse → ETL → stage" step from the API & SSOR Data Ingress concept — everything up to and including the staging landing.

**What we evaluated.** Seven ways to build it, across three families: **Azure managed** (ADF, Databricks, Fabric), **Azure DIY** (roll-your-own from serverless primitives), and **off-Azure** (Oracle OCI-native, on-prem Oracle DB, on-prem SQL Server). Each was costed bottoms-up with **license, infrastructure, and labor as distinct streams** (capex + monthly), on live Azure retail prices and 2026 Oracle/Microsoft license list prices.

**What we found.** The three managed Azure options are a **band, not a ranking**: ADF ~$159k, Fabric ~$203k, Databricks ~$231k fall inside the labor-estimate noise (labor is 63–90% of each), with ADF at the low edge — so we do not stand behind the precise order within the trio. The **decisive, cross-family** gaps do survive the assumptions: managed band (~$159k–$231k) « **Oracle OCI-native BYOL ~$233k** (the like-for-like figure, since the org already owns Oracle DB licenses; Lic-Incl is $256k) « roll-your-own ~$388k « on-premise ~$506k (Oracle Enterprise Edition reaches $763k). Separating license from infrastructure is what makes this legible: license is an explicit SKU off-cloud but bundled into hyperscaler rates (Fabric ~41% license; roll-your-own only 2%); on-premise cost is hardware (~$210k/3yr) + labor (~$162k/3yr), not license — until Oracle EE, whose license alone (~$315k) exceeds the entire ADF option. **Two contingencies sit outside these figures, both favoring OCI-native:** a conditional **GoldenGate** line if telematics needs low-latency CDC (~$8k–58k/3yr; applies to every Azure approach, $0 for OCI-native — the default batch landing needs none), and cross-cloud **egress** (immaterial in dollars, but the one line OCI-native zeroes).

**Scope note.** Whichever approach builds it, this tier lands in **Oracle EDW staging**; Power BI consumption is downstream of aiWorks on the Oracle EDW and identical across all seven approaches. So BI-platform "nativeness" (Fabric/Power BI) is **not** a selection factor — Fabric is judged on cost, Oracle-sink maturity, and capacity only.

**Recommendation.** Build on **ADF** — at the low edge of the managed band with the lightest build and near-zero *engine* upkeep, and the most mature Oracle sink. Design the landing contract and lake so heavy/streaming workstreams can **graduate to Databricks** incrementally. Treat **Fabric** as a cost-only contender (its F8-reserved best case is the cheapest *if* it tunes to capacity) validated by a scoped PoC — not as an ecosystem pick. **Oracle OCI-native (BYOL ~$233k)** is the strongest non-Azure choice: it removes the cross-cloud hop and the landing step entirely; prefer Azure over it on **workload isolation** (don't run ETL on the ADW that serves reporting) and connector breadth, not on cost, where the ~$30k gap is not decisive. **On-premise** is justified only by a hard data-residency/sovereignty mandate or a large existing on-prem investment.

**Next step.** A 2–3 week PoC landing two contrasting feeds end-to-end into OCI staging — GeoTab telematics (Lane 1) **via the intended streaming/CDC path, to answer the GoldenGate question** + FastTrak or CGI Advantage (Lane 3) via P2 — with the QUAR_/manifest hand-off to Agiline's aiWorks load (which runs Billow's PL/SQL). It must return the telematics landing cost and real build/ops hours, converting the cost band into committed numbers and de-risking the platform choice with data; run a Fabric F-SKU trial in parallel on cost grounds only.

*All figures are planning estimates; enterprise license discounts (50–80% off list) and cloud reservations (30–41% off) are not applied. Reproduce via `../alternates/seven_approach_cost_model.py`.*
