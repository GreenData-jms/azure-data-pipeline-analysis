# Cost model — `erc_azure_cost_model.py`

A reproducible, bottoms-up cost model for the four Azure ingress options. Unit prices are **live Azure Retail Prices** (region West US, PAYG Consumption, pulled 2026-07-22); volumes are **documented, editable assumptions**.

> **Full assumptions & basis:** [`ASSUMPTIONS.md`](ASSUMPTIONS.md) — every data-volume, compute (with SKUs), storage, bandwidth, and labor assumption behind the numbers, a worked Variant-A build-up, the load-bearing-vs-rounding-error read, and the off-Azure sizing. Feeds the deck's basis & appendix slides.

## Run it

```bash
python3 erc_azure_cost_model.py
```

No dependencies (standard library only). Prints per-option monthly buildups, the four-way run-vs-run+ops comparison, one-time build, 3-year TCO, sensitivity (0.5×/1×/2×), and the DIY verdict.

## Reprice it

Edit the two dicts at the top:

- **`P`** — unit prices. Refresh from the Azure Retail Prices API if rates change, or switch region.
- **`A`** — volume & labor assumptions: `sources`, `activity_runs_mo`, `steady_ingest_gb_mo`, `telematics_events_mo`, `transform_cluster_hours_mo`, `blended_rate`, `build_hours`, and **`ops_hours_mo`** (the load-bearing DIY assumption).

Then re-run and update `corpus/figures.json` to match.

## The two assumptions that move the answer most

1. **`ops_hours_mo["RollYourOwn"]`** — ongoing maintenance for the roll-your-own option. At 40 h/mo it dominates DIY's TCO; even at 20 h/mo DIY stays above ADF.
2. **`telematics_events_mo`** — drives Databricks' streaming line and Fabric's capacity tier.

## Output ↔ figures.json

Every number in `corpus/figures.json` and the proposals is generated by this script. If you change assumptions, treat the script as the source and regenerate the JSON.

## Caveats

Planning estimates, not a quote. Reservations/savings plans (1–3 yr, −30–41% on compute) are **not** in the PAYG baseline. Actuals depend on final volumes, cadence, and cluster/capacity tuning — a PoC converts these to committed numbers.
